Installation

Global Install (Recommended)

npm install -g project-kickoff

After installation, the kickoff command is available globally.

Using npx (No Install)

npx project-kickoff create my-app

Run directly without installing. Perfect for one-off use.

Quick Start

Create a new project
Interactive wizard guides you through options
kickoff create my-app
Or use a preset for instant setup
Skip the wizard with pre-configured stacks
kickoff create my-app --preset saas-starter
Install dependencies and start
Your project is ready to develop
cd my-app && npm install && npm run dev

Interactive Wizard

When you run kickoff create my-app without a preset, the interactive wizard guides you through building your perfect stack.

How It Works

Step-by-Step Configuration

The wizard asks questions in logical order, with smart defaults and conditional follow-ups:

  1. Preset or Custom? - Choose a preset for quick setup, or "Custom" to configure everything
  2. Complexity Track - Quick (minimal), Standard (full features), or Production (enterprise-ready)
  3. Project Type - Next.js, FastAPI, Hono, Rust, Go, CLI, MCP server, etc.
  4. Runtime - Node, Bun, Deno, Python, Go, or Rust (auto-detected from project type)
  5. Database - Supabase, Neon, Turso, PostgreSQL, SQLite, etc.
  6. ORM - Drizzle, Prisma, SQLAlchemy, GORM, SQLx (language-appropriate options)
  7. Authentication - Clerk, Better Auth, Supabase Auth, Auth0, etc.
  8. AI/ML (optional) - AI framework, vector DB, embeddings, local LLM

Smart Conditional Questions

Questions adapt based on your previous answers:

  • → Choose Supabase as database → Supabase Auth appears as recommended auth option
  • → Choose FastAPI project → Shows Python ORMs (SQLAlchemy, SQLModel, Tortoise)
  • → Choose Convex/PocketBase/Firebase → ORM question is skipped (BaaS handles it)
  • → Enable AI Framework → Vector DB and Embedding questions appear
  • → Choose Quick complexity → Auth and AI questions are skipped

Wizard Options

OptionDescription
-y, --yes Accept all default values (non-interactive mode)
-p, --preset <name> Skip wizard entirely, use a preset configuration
--dry-run Show what would be created without writing files
-v, --verbose Show detailed logging during project creation

Complexity Tracks

Quick Track

Minimal setup for prototypes and experiments. Skips auth, AI questions, and production config.

  • 5-minute setup
  • File-based storage (SQLite/JSON)
  • No external dependencies required
Standard Track

Full-featured setup for real applications. Includes auth, AI options, and production-ready defaults.

  • 15-minute setup
  • Real database with ORM
  • Authentication included
  • AI/ML optional
Production Track

Enterprise-grade setup with security, observability, and compliance considerations.

  • 30-minute setup
  • Monitoring and observability
  • CI/CD pipelines included
  • Security best practices
  • Web server configuration (Caddy/nginx/Traefik)

Common Workflows

SaaS Application

Next.js + Supabase + Drizzle ORM
kickoff create my-saas --preset saas-starter
cd my-saas
npm install
# Add your Supabase keys to .env
npm run dev

Includes: Authentication, database, design system (shadcn/ui), Docker, CI/CD

AI/RAG Application

Next.js + Supabase Vector + Vercel AI SDK
kickoff create my-ai-app --preset ai-rag-app
cd my-ai-app
npm install
# Add OPENAI_API_KEY and Supabase keys to .env
npm run dev

Includes: Vector search, embeddings, streaming AI responses, RAG pipeline

Local AI Agent (No Cloud Costs)

Hono + Ollama + LangChain
# First install Ollama: https://ollama.ai
ollama pull llama3

kickoff create my-agent --preset ai-agent
cd my-agent
bun install
bun run dev

Includes: Local LLM inference, Chroma vector DB, LangChain agents

Python API

FastAPI + PostgreSQL + SQLAlchemy
kickoff create my-api --preset fastapi-starter
cd my-api
uv sync  # or: pip install -r requirements.txt
uvicorn main:app --reload

Includes: Async API, auto OpenAPI docs, database migrations, Docker

Go Microservice

Gin + PostgreSQL + GORM
kickoff create my-service --preset go-microservice
cd my-service
go mod tidy
docker compose up -d db
go run main.go

Includes: High-performance API, connection pooling, structured logging

Rust API

Axum + PostgreSQL + SQLx
kickoff create my-rust-api --preset rust-api
cd my-rust-api
docker compose up -d db
cargo run

Includes: Compile-time SQL verification, async runtime, maximum performance

Edge API

Hono + Turso + Bun
kickoff create my-edge-api --preset edge-api
cd my-edge-api
bun install
# Add TURSO_DATABASE_URL and TURSO_AUTH_TOKEN to .env
bun run dev

Includes: Edge-first, global SQLite, deploy to Cloudflare/Deno/Vercel

Troubleshooting

Error: Directory already exists

The project name you specified already exists as a directory.

Solution

Choose a different name or delete the existing directory:

rm -rf my-app  # Be careful!
kickoff create my-app
Node.js version error

Kickoff requires Node.js 18 or higher.

Solution

Update Node.js using nvm or download from nodejs.org:

nvm install 20
nvm use 20
node --version  # Should show v20.x.x
Git initialization failed

Git is not installed or not in PATH.

Solution

Skip git init or install git:

# Skip git initialization
kickoff create my-app --skip-git

# Or install git
# macOS: brew install git
# Ubuntu: sudo apt install git
Port 3000 already in use

Another application is using the default port.

Solution

Kill the process or use a different port:

# Find and kill the process
lsof -i :3000
kill -9 <PID>

# Or change port in your project
PORT=3001 npm run dev
Template not found

The specified project type or preset doesn't exist.

Solution

List available options:

kickoff list  # See all presets
Permission denied errors

npm global install requires elevated permissions.

Solution

Fix npm permissions or use npx:

# Option 1: Use npx (no install needed)
npx project-kickoff create my-app

# Option 2: Fix npm permissions
mkdir ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
